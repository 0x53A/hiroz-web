zenoh & rust in wasm

Eclipse zenoh and hiroz (ros-z) compiled to WebAssembly with a SharedArrayBuffer threadpool — a full zenoh client and a ROS 2 node, running in your browser tab.

hiroz ↔ ROS 2
Interactive: connect to a zenoh router, watch the live /chatter feed from a ROS 2 talker (rmw_zenoh), publish your own std_msgs/String messages. zenoh threaded runtime test
Diagnostics: 6 runtime tests (cross-worker spawn, block_in_place, session open, pub/sub roundtrip) against a router endpoint of your choice.

Both demos need a zenoh router reachable from your browser, e.g. locally:
zenohd --listen ws/0.0.0.0:7448 → endpoint ws/127.0.0.1:7448.
From this https page, plain ws/ works only for 127.0.0.1/localhost; remote routers need wss/. For the ROS 2 side, see the docker compose stack.
